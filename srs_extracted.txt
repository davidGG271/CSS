presente documento de Especificación de Requisitos de Software (SRS) tiene como objetivo describir las funciones y características del sistema de venta y personalización de computadoras desarrollado para mejorar la gestión de productos, pedidos y clientes. El sistema permitirá a los usuarios registrarse, iniciar sesión, realizar compras y armar computadoras personalizadas según sus necesidades, mientras que los administradores podrán gestionar inventarios, pedidos y notificaciones. Está dirigido al equipo de desarrollo, docentes y evaluadores del proyecto, permitiendo comprender el funcionamiento y los objetivos principales del sistema.  `n               1.2 Alcance        `n             El sistema consiste en una plataforma de venta y personalización de computadoras orientada al mercado peruano, donde los usuarios podrán registrarse, iniciar sesión, comprar productos y armar PCs según sus preferencias. El objetivo principal es mejorar la gestión de inventario, pedidos y clientes, además de optimizar la experiencia de compra. El sistema estará enfocado únicamente en la administración de productos, ventas y usuarios dentro de la plataforma. Entre los beneficios esperados destacan una mejor organización de datos, mayor eficiencia operativa, reducción de errores en inventario y una comunicación más efectiva con los clientes mediante notificaciones y correos electrónicos automatizados.       `n            2. Descripción General   `n           2.1 Perspectiva del Producto   `n             El sistema será una plataforma independiente orientada a la venta y personalización de computadoras y componentes electrónicos. Su función principal será gestionar usuarios, productos, pedidos, inventario y notificaciones dentro de una misma plataforma. Además, el sistema tendrá relación con una base de datos para almacenar información de clientes, productos y compras, así como con un servicio de correo electrónico para el envío de notificaciones y anuncios. Los límites del sistema comprenden únicamente los procesos relacionados con el registro de usuarios, gestión de productos, compras y administración interna, sin incluir servicios externos de logística o distribución física de productos.        `n           2.2 Funciones del Producto   `n         1. Para el inicio de sesión, contará con dos campos de entrada de texto (Correo y Contraseńa), un enlace para "Recuperar contraseńa" y un botón principal de color contrastante con el texto "Iniciar Sesión". Para el registro, la vista cambiará a un formulario extendido que incluirá validaciones visuales en tiempo real (por ejemplo, el borde del campo de texto se pondrá rojo si el DNI no tiene los 8 dígitos requeridos).  `n                                                     `n                                                     `n               `n                 3.1.2 Búsqueda y visualización de productos:  `n                 Este proceso se realiza principalmente mediante la pantalla principal, el catálogo general, las categorías y la vista de detalle del producto. En la pantalla principal se presentan accesos rápidos al catálogo, categorías destacadas y productos recomendados. El catálogo muestra los productos disponibles en formato de tarjetas, organizadas en una grilla visual que incluye nombre, marca, precio, stock y botón para agregar al carrito. Además, el usuario puede navegar por categorías como componentes, periféricos, computadoras, notebooks, monitores y gaming, donde se muestran únicamente los productos correspondientes. Al seleccionar un producto, se accede a una pantalla de detalle con información más completa, como descripción, especificaciones, disponibilidad, precio y selector de cantidad. Desde esta pantalla el usuario puede agregar el producto al carrito, continuando así con el proceso de compra.  `n                                                     `n                                                     `n                                                         `n                 3.1.3 Carrito de compras y pedidos:  `n                 El proceso de carrito y pedidos inicia cuando el usuario agrega un producto desde el catálogo o desde la vista de detalle. La pantalla del carrito muestra los artículos seleccionados en tarjetas horizontales con nombre, marca, precio, cantidad y controles para aumentar, disminuir o eliminar productos. Junto a la lista de productos se presenta un resumen de compra con subtotal, costo de envío y total, además del botón para continuar al pago. Al ingresar al checkout, la interfaz se divide en pasos: primero se solicitan los datos de envío del cliente y luego se selecciona el método de pago. Durante este proceso se mantiene visible un resumen del pedido para que el usuario confirme los productos y montos antes de pagar. Finalmente, al confirmar la compra, se muestra una pantalla de confirmación y el pedido queda registrado para poder consultarse posteriormente desde la sección de cuenta, donde se listan los pedidos realizados con su estado, fecha, productos y monto total.  `n                                                     `n               `n               `n               `n               `n               `n                    `n                 3.1.4 Gestión y Configuración de PCs para venta:  `n                 La creación de computadoras pre-configuradas se realiza desde la sección administrativa denominada "PCs Armadas". La interfaz se divide en dos áreas principales. Primero, una vista de catálogo basada en una cuadrícula de tarjetas visuales que muestra las configuraciones ya existentes, exhibiendo su imagen principal, los componentes clave incluidos, el precio total calculado, el stock disponible y un botón de eliminación (papelera). Segundo, un formulario estructurado accesible mediante el botón "Nueva Build". Este formulario presenta campos de texto para asignar un nombre y la URL de la imagen, seguidos de una lista interactiva de selectores desplegables. Estos selectores permiten al administrador elegir piezas específicas directamente del inventario real (Procesador, Motherboard, Memoria RAM, Tarjeta gráfica, Almacenamiento, Fuente de poder, Case y Refrigeración). A medida que se seleccionan los componentes, el sistema calcula de forma dinámica el precio total del equipo. Finalmente, un botón de "Guardar PC Armada" permite almacenar la configuración en la base de datos para integrarla inmediatamente al catálogo de la tienda.  `n                                                     `n               `n               `n               `n                                                     `n                                                     `n                 3.1.5 Gestión de pedidos:    La interfaz de gestión de pedidos está diseńada para proporcionar un control exhaustivo sobre el flujo de ventas, presentándose como una vista de cuadrícula de datos a pantalla completa bajo la temática visual "Dark-Tech" del panel administrativo. En la parte superior, la interfaz cuenta con una tarjeta de herramientas que incluye una barra de búsqueda de texto libre (para localizar órdenes por nombre o ID) y un menú desplegable para filtrar dinámicamente los registros según su estado logístico. La tabla principal expone cada transacción en filas individuales, detallando de forma clara el ID, Nombre del Cliente, Fecha, Total calculado y Método de pago empleado. En la última columna de cada fila se integra directamente un selector de estado interactivo; este elemento permite al administrador modificar la fase del despacho (Pendiente, Pagado, En preparación, Entregado, Cancelado) sin tener que abrir ventanas adicionales, brindando retroalimentación visual inmediata mediante alertas emergentes que confirman si la actualización en la base de datos fue exitosa.  `n                                                     `n                                                     `n                                                     `n                                                     `n                                                     `n               `n                     `n           3.2 Requisitos Funcionales (User Story)   `n             Anote cada User Story con un identificador único (US-XX), descripción, prioridad y criterios de aceptación.        `n                 Tabla Resumen de User Story:       `n                                                       ID        `n                           User Story        `n                          Prioridad (Alta/Media/Baja)        `n                           Criterios de Aceptación        `n                                 US-01        `n                       Como usuario, quiero registrarme en la plataforma para acceder al sistema de compras.        `n                       Alta        `n                     -
